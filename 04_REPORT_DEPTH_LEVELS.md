# Report Depth Levels

## Quick

Use when the user wants a fast view.

Target length: 1 page or less.

Sections:

- Bottom line
- Key evidence
- Main risk
- Confidence
- What to verify next

## Standard

Use as the default.

Target length: 2-4 pages.

Sections:

- Executive summary
- Company snapshot
- Financial trend
- Valuation view
- Catalysts
- Risks
- Base, bull, bear case
- Final view

## Deep

Use when the user asks for detailed research.

Target length: 5-12 pages.

Sections:

- Executive summary
- Business model
- Segment analysis
- Financial statement trends
- Industry and peer comparison
- Management guidance and events
- Valuation range
- Scenario analysis
- Risk review
- Investment conclusion
- Data appendix

## Expert

Use when the user needs modeling, portfolio work, or investment committee depth.

Target length: variable.

Possible modules:

- full financial model,
- DCF assumptions,
- peer comp table,
- factor exposure,
- backtest,
- portfolio construction,
- sensitivity tables,
- source appendix.

## Compression Rule

Higher depth should mean more evidence, not more filler.

Each additional section must earn its place.

## Language Modes

Reports should eventually support user-selectable language modes:

- Chinese
- English
- bilingual Chinese-English

For now, language selection is a planning requirement, not a separate implementation layer.

Default behavior:

- match the user's language unless the user requests otherwise,
- preserve financial terms clearly,
- avoid awkward literal translation,
- keep bilingual output concise rather than duplicating unnecessary filler.
