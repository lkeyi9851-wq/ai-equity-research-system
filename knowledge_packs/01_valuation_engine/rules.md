# Valuation Engine Rules

Highest principle: Less is More.

Execution rule: no valuation claim without explicit inputs, formula, and reconciliation.

Rules:
- Only use real inputs, explicit assumptions, and explainable formulas.
- If data is missing, write data gap. Do not invent valuation cases.
- Bear/Base/Bull must be real cases, not cosmetic multiple changes.
- Target price and DCF from external reports are market-view references, not standalone truth.
- If valuation inputs conflict, rating confidence must fall unless the conservative case still supports the rating.
- If price is stale or non-live, final confidence cannot be High unless valuation gap is very large and clearly labeled.
- Current valuation must be recalculated before it is interpreted.

Failure blocked:
- A memo cannot say Buy because a source reports attractive P/FCF while internally recalculated P/FCF says something different.
