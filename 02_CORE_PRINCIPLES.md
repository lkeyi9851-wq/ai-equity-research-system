# Core Principle

## Less Is More

Less is more is the highest principle of this project.

It means the system should reach the clearest useful judgment with the fewest necessary moving parts.

It does not mean shallow work. It means deep work, compressed.

## Execution Rule

No thesis without transmission.

Every retained claim in the final memo must carry a decision chain:

```text
source evidence
-> business mechanism
-> financial line item
-> market expectation gap
-> valuation / rating impact
-> invalidation trigger
```

If the chain breaks, the claim stays in source notes or is removed.

Less is more controls output length. Transmission controls information density.

## What It Means

### 1. Fewer Inputs

The user should be able to give a short natural request. The system should infer reasonable defaults and ask only when the missing information would materially change the work.

### 2. Fewer Agents

Each agent must prevent a real failure mode. Do not add an agent because the architecture looks more advanced with another box.

### 3. Fewer Data Points

Prefer the smallest reliable evidence package that can support the selected depth. One strong source is better than five weak ones.

### 4. Fewer Words

Reports should be as short as possible while still preserving truth, uncertainty, and decision value.

Short output is earned before writing. Do the filtering internally; show only the conclusion, the key evidence, the invalidation condition, and confidence.

### 5. Fewer Repeated Efforts

Reuse validated context. Update only what changed. Keep reusable industry knowledge compact, sourced, and dated.

### 6. Fewer False Certainties

When the evidence is incomplete, say so. A concise uncertain answer is better than a confident polished one built on weak data.

## Operating Test

Before adding data, agents, charts, sections, or automation, ask:

```text
Will this materially improve the decision?
```

If not, leave it out.

## Equity Research Test

For stock research, "decision value" means one of four things:

- changes the future cash-flow view,
- changes growth durability or margin quality,
- changes downside risk,
- changes the valuation multiple or fair-value range.

Everything else is background.

## Compression Rule

The final memo should not contain:

- visible self-checks,
- generic caveats,
- source summaries that do not support a thesis,
- repeated facts,
- analyst opinions without the assumptions behind them.

Keep those in source notes if they are useful for audit.
